Experiment 1:   SVG Animation

I started with demo from w3schools. The link to it is in references below. This demo does a move, rotate and scale text. It also grows and changes color of a rectangle.

This demo embeds the animation in the SVG. It is able to animate components of the SVG seperately. The code snippet below shows how this is accomplished.

  •         <animate attributeName="x" attributeType="XML" begin="0s" dur="6s" fill="freeze" from="250" to="0" />
  •         <animate attributeName="y" attributeType="XML" begin="0s" dur="6s" fill="freeze" from="100" to="0" />

Experiment 2:   CSS Keyframe Animation

My main experiment was to figure out how to duplicate this effect in CSS keyframes. I drew the pictures in seperate SVG files.

The code snippet below shows how simple this task is in HTML. The coplexity is in the keyframes.

One thing to notice is the ability to change colors in the embedded SVG animation. This cannot be done in the CSS version. CSS deals with the entire object and cannot modify its internal properties.

Another difference is the timing. All values are set the same but it appears that the CSS version completes sooner. This is true in both Chrome and IE. Actually they rectangle takes the same amount of time to zoom in in both cases. In the embedded version it changes the absolute size linearly. In the CSS it zooms the percent size linearly. This makes the absolute size change non-linear. Notice they both actually finish at the same time but the CSS version slow down as it approached the final point.

Animation with CSS Keyframes

Note in the keyframe for the box I used the words "from" and "to" for the frame designation. "0%" for "from" and "100%" for "to" can be used interchangeably. In this application since I only need 2 key frames then it is clearer to use the words instead of the numbers.

  •         animation: myBoxAnimate 6s forwards;

Experiment 3:   In Internet Explorer

SVG graphics are animated internally by a spec called SMIL (Synchronized Multimedia Intergration Language). SMIL like SVG are XML based.

Internet Explorer does not support this spec at this time. IE requires a third party script to support annimation embeded in an SVG. The most popular one is FakeSmil. It was started in 2008 and mostly complete soon after. It is only a partial implemtation and is missing several features of the standard and only partially implements others. Other than a few bug fixes little has been done on it for several years. Mostly because all browsers now support SMIL natively. Internet Explorer is the last remaining major browser that does not.

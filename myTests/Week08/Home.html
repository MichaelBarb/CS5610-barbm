Summary

This set of experiments continues from last week refining my Home page. I was pleased with the results but felt some important improvements needed to be made to the carousel to make it look more professional.

There is still one thing about the responsive nature and the carousel that need to be dealt with. As the page narrows the carousel slides get longer. They do not all end up as the same length. Even if the carousel is not visible the page jumps up and down as the slide changes. Some research shows this to be a difficult problem.

Experiment 1:   Carousel - Add Picture

The first improvement was to add a larger picture of myself to the carousel. The one in the navbar is small but OK for this location. Using one of Bootstraps classes I made the picture an oval. The shape provides a more distinctive appearance.

  • <img src="../../myImage/myPicture.jpg" alt=""
  •    class="img-circle center-block"
  •    style="width:154px;height:180px" />

Experiment 2:   Carousel - Add Grid

Just adding the picture did not provide a balanced appearance to the slide especially the responsive results. I used Bootstraps grid to provide a more pleasing appearance. Dividing up the 12 units as follows:

  • 2 wide blank column
  • 3 wide picture column
  • 7 wide title column
  • 2 wide blank column

For the full wide page the blank columns provide better centering. In the responsive behavior of the page as you narrow it the blank columns disappear first.

The 3 wide picture column provides room for the picture without it overlapping the text. When the page gets narrow this picture will center itself above the titles.

The following code snippet is the final result:

  • <div class="item container">
  •     <div class="row">
  •         <div class="col-sm-2"></div>
  •         <div class="col-sm-3">
  •             <img src="../../myImage/myPicture.jpg" alt=""
  •                 class="img-circle center-block"
  •                 style="width:154px;height:180px" />
  •         </div>
  •         <div class="col-sm-7">
  •             <h1>Michael Barb</h1>
  •             <p class="lead">Master's Science Computer Science Candidate</p>
  •         </div>
  •     </div>
  • </div>

Experiment 4:   Carousel - Indicator Colors

The small dots in the carousel that let you know which slide you are on and let you got a slide had very little contrast to the background. After some research I found that I needed to add some styles to myTheme.css file to change the colors. Following are the lines that were added:

  • .carousel-indicators li {
  •     border-color:slategray;
  • }
  • .carousel-indicators .active {
  •     background-color: slategray;
  • }

Experiment 5:   # Jumps

The fixed navbar the top actually overlays the page. When you do # jumps to locations on the page they end up under the navbar and it looks like the jump is not correct. Many help formums on the web tell you change padding for the container. I found this did not help. I think it may have worked in older versions of Bootstrap but not in the latest one. Eventually I found a reference to tapping in to an event “hashtagchange”. This worked very well and I added a JavaScript file to the page. Following is the JavaScript code:

  • window.addEventListener("hashchange", function () {
  •     scrollBy(0, -60);
  • });

Experiment 6:   SVG Images

To illustrate Experiment 3 I need a quick diagram of the nested grid. So it would scale I wanted use an SVG picture. I found an open source program called Inkscape. It let me quickly draw the image for the nested grid. The link to the program is in references. This is the line that called the diagram that I drew:

  • <img src="../../myImage/NestedGrid.svg"
  •     class="img-responsive center-block"
  •     width="310" height="110" />

UPDATE: It seems our file serve will not serve svg image files. I hard coded the diagram in the page. When doing this the diagram is no longer responsive. If I am able to get our server to updated I will revert back to the svg image.

In the previous set of experiments HTML help from w3schools they recommended against doing the width and height the way I have it above. They recommend using styles for this. I found using styles interferes with img-responsive. Only the width changes and not the height which changes the aspect ratio and distorts the image. Putting it the way I have it above let's Bootstrap change both and maintain the aspect ratio.

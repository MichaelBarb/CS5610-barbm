Experiment 2:   BootStrap Panel and Well

In reading about Bootstrap I noticed they had sytles Panel and Grid. For my code snippet box I wondered if I could have cleaner code using one of them.

I also wanted a better responsive result. I used Bootstraps grid for this.

Panel is really intended to be a two part box. See example below. I could force it to be a single part part. I could force it to be a single color but it still reqired overrides on the ul text. Panel also has some neat ways to hide the panel content but for this application it is not needed.

Panel Info With Title

Panel content

Well is suppose to provide a 3D inset look. I found its default colors and shape do not really work for this. In other programs I have worked with the effect was much better.

I was able to develope a style that would override most default settings and give me the look I wanted. Using Well did not really gain anything. It required styling also most the same as what I had done before and just added a CSS style that was mostly overridden.

Experiment 3:   Update myCSS snippetBox

Experimenting with styles for for the Well made realize I could do a much better job on my previous style. I used a combinator style so once I set the style for the division it set the text style for the "ul" inside it.

The thing that made the responsive behavior better was putting the snippet box in a grid and getting rid of a lot of padding in the style. I used a small grid that was 1, 10, 1. This made the empth cells on each end dissappear at 750px. It also made nice consistant bumps in cell size as the width changed.

  • .snippetBox2 {
  •     padding: 10px;
  •     border: solid 3px black;
  •     background-color: #d0e0ff;
  •     margin-left: auto;
  •     margin-right: auto;
  • }
  • .snippetBox2 ul {
  •     font-family: monospace;
  •     font-size: 80%;
  •     margin: 2px;
  • }
